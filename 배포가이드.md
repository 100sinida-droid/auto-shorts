# 한강 수온 대시보드 — 배포 가이드 (깃허브 한 곳 + Cloudflare 자동 연동)

이제 **깃허브 저장소 하나**에만 올리면 됩니다.
Cloudflare Pages를 저장소에 연결해두면, 앞으로는 **깃허브에 push할 때마다 Cloudflare가 자동으로 새로 배포**합니다. Wrangler 명령어도, 주소 복사·붙여넣기도 필요 없어요.

## 폴더 구조 (이대로 저장소에 올리면 됩니다)

```
(저장소 루트)
├── index.html              ← 화면
└── functions/
    └── api/
        ├── quotes.js        ← 자동으로 /api/quotes 가 됨 (시세)
        └── chat.js          ← 자동으로 /api/chat 이 됨 (채팅)
```

`functions/` 폴더는 Cloudflare Pages가 자동으로 인식해서 백엔드로 만들어 줍니다.
화면과 백엔드가 **같은 주소**를 쓰므로 설정할 API 주소 같은 것도 없습니다.

---

## ① 깃허브에 올리기

1. 깃허브에서 새 저장소를 만듭니다 (예: `hangang`).
2. 이 폴더의 내용물(`index.html` 과 `functions` 폴더)을 그대로 올립니다.
   - 웹에서 하려면: 저장소 → **Add file → Upload files** → 폴더째 끌어다 놓기 → Commit.
   - `functions` 폴더 구조(안의 `api/quotes.js`, `api/chat.js`)가 그대로 유지되어야 합니다.

여기까지가 깃허브에서 할 일의 전부입니다.

---

## ② Cloudflare Pages에 저장소 연결 (최초 1회만)

1. https://dash.cloudflare.com 접속 → 왼쪽 메뉴 **Workers & Pages** → **Create** → **Pages** 탭 → **Connect to Git**.
2. 방금 만든 깃허브 저장소를 선택합니다. (처음이면 깃허브 계정 연결/권한 허용을 한 번 합니다.)
3. 빌드 설정은 **비워두면 됩니다**:
   - Framework preset: **None**
   - Build command: **(비움)**
   - Build output directory: **/** (또는 비움)
4. **Save and Deploy** 클릭.

1~2분 뒤 `https://<프로젝트이름>.pages.dev` 주소가 생성됩니다. 접속하면 화면과 시세가 바로 나옵니다.

> 시세(`/api/quotes`)는 이 단계까지만 해도 바로 작동합니다. 채팅만 아래 ③ KV 연결이 추가로 필요합니다.

---

## ③ 채팅·방문자 켜기 — 둘 중 하나 선택

채팅과 방문자 수는 저장소가 필요합니다. **방법 A(JSONBin)가 더 쉬워서 추천**해요. 대시보드 클릭이 익숙하면 방법 B(KV)도 됩니다.

### 방법 A · JSONBin (권장, 5분, 대시보드 설정 없음)

1. https://jsonbin.io 가입(무료) → 로그인.
2. 우측 위 프로필 → **API Keys** → **X-Master-Key** 복사.
3. JSONBin에서 **Create Bin** 두 개 생성:
   - 채팅용: 내용에 `{"messages":[]}` 넣고 저장 → 주소창/목록의 **Bin ID** 복사
   - 방문자용: 내용에 `{"days":{},"total":0}` 넣고 저장 → **Bin ID** 복사
4. `index.html` 상단의 이 부분을 채웁니다:
   ```js
   const JSONBIN_KEY = "여기에_X-Master-Key";
   const CHAT_BIN    = "채팅용_Bin_ID";
   const VISIT_BIN   = "방문자용_Bin_ID";
   ```
5. 깃허브에 push → Cloudflare가 자동 재배포 → 채팅·방문자 즉시 작동.

> 세 값을 비워두면 자동으로 방법 B(KV)로 넘어갑니다.

### 방법 B · Cloudflare KV (대시보드 설정)

1. **Workers & Pages → KV → Create a namespace** (이름 예: `hangang-chat`).
2. Pages 프로젝트 → **Settings → Bindings → Add binding**
   - Variable name: **CHAT** (대문자)
   - KV namespace: 방금 만든 것 선택 → 저장
3. **Retry deployment**(또는 커밋 push)로 재배포.

> 이 KV 하나로 채팅과 방문자 카운터가 함께 작동합니다.

---

## ④ 실제 한강 수온 (추가 설정 없이 작동)

한강 수온은 **hangang.life 공개 API**(무인증)에서 탄천·중랑천·안양천·선유 자동측정소의 실측값을 가져옵니다. **인증키가 필요 없고, ②번 배포만 끝나면 바로 나옵니다.**

- 확인: `https://<프로젝트>.pages.dev/api/watertemp` 를 열어 `{"ok":true,"avgTemp":...,"stations":[...]}` 가 나오면 성공.
- 화면 수온 아래에 "실측 수온"과 측정소별 값이 표시됩니다.

> (선택) 백업 소스로 서울시 열린데이터광장도 넣어뒀습니다. hangang.life가 죽을 때를 대비해 서울시 인증키를 쓰고 싶으면, data.seoul.go.kr에서 무료 인증키를 발급받아 Cloudflare Pages → Settings → Environment variables 에 `SEOUL_API_KEY` 로 추가하면 됩니다. 없어도 정상 작동합니다.

---

## 앞으로 수정할 때

`index.html` 이나 `functions/` 안의 파일을 고쳐서 깃허브에 push만 하면,
Cloudflare가 알아서 다시 배포합니다. 대시보드를 다시 만질 필요가 없습니다.

## 체크리스트

- [ ] `https://<프로젝트>.pages.dev/api/quotes` 를 열면 JSON 시세가 나온다
- [ ] 사이트 상단 배지가 "실시간"으로 바뀌고 코스피/종목 값이 채워진다
- [ ] 유튜브 한강 라이브가 재생된다 (https라서 정상)
- [ ] (KV 연결 후) 채팅을 보내면 남고, 새로고침·다른 기기에서도 보인다
- [ ] (수온 키 연결 후) `/api/watertemp` 가 `"ok":true` JSON을 반환하고, 화면에 "실측 수온"이 표시된다

## 자주 나는 문제

**시세 자리에 "백엔드(/api/quotes)가 아직 켜지지 않았어요"가 뜸 (가장 흔함)**
이건 화면은 떴는데 `functions/` 폴더가 백엔드로 인식되지 않은 상태입니다. `/api/quotes`가 JSON 대신 화면(index.html)을 돌려주고 있는 거예요. 아래를 확인하세요:
1. 저장소 **최상단**에 `index.html`과 `functions/` 폴더가 나란히 있어야 합니다. `functions`가 하위 폴더(예: `site/functions`) 안에 들어가 있으면 인식이 안 됩니다.
2. Cloudflare Pages 프로젝트 → **Settings → Builds & deployments → Build configurations** 에서 **Root directory**가 비어 있거나 `/` 인지 확인하세요. 여기에 `site` 같은 하위 경로가 적혀 있으면 `functions`를 못 찾습니다.
3. 위를 고친 뒤 저장소에 커밋을 한 번 push하거나 **Retry deployment**로 재배포하세요.
4. 확인: 브라우저에서 `https://<프로젝트>.pages.dev/api/quotes` 를 직접 열어 `{"kospi":...}` JSON이 나오면 정상입니다.

**시세 조회 실패 (HTTP 오류)**
→ 야후가 일시적으로 응답을 안 준 경우입니다. 새로고침 버튼을 다시 눌러보세요.

**채팅을 보내면 뜨는데 새로고침하면 사라짐 / 방문자 수가 "--"**
→ KV(CHAT)가 연결되지 않은 상태입니다. 메시지가 저장되지 않고 화면에만 잠깐 보이는 거예요. ③번을 따라 KV 네임스페이스를 변수명 **CHAT**(대문자)으로 연결하고 재배포하세요. `https://<프로젝트>.pages.dev/api/chat` 를 열어 `"kv":true` 면 정상, `"kv":false` 면 아직 미연결입니다. 방문자 카운터도 같은 KV를 쓰므로 함께 켜집니다.

**코스피 값이 몇 분 지연됨**
→ 정상입니다. 야후 데이터 + 30초 캐시라 초 단위 실시간은 아닙니다.

**유튜브 영상이 안 나옴**
→ 이 앱은 특정 영상이 아니라 **채널의 현재 라이브**를 자동 재생하므로 방송 영상이 바뀌어도 유지됩니다. 그래도 안 보이면 해당 채널이 잠시 방송을 쉬는 중일 수 있어요. 영상 위 탭에서 다른 채널로 바꾸거나, 아래 링크로 채널에서 직접 여세요. (반드시 pages.dev 주소로 여세요. 로컬 file:// 로 열면 재생이 막힙니다.)

**수온이 "실측 조회 실패 / 추정치"로 나옴**
→ `https://<프로젝트>.pages.dev/api/watertemp` 를 직접 열어보세요. `"ok":true`가 나오는데 화면만 추정치면 브라우저 캐시 문제이니 강력 새로고침(Ctrl+F5) 하세요. `"ok":false`면 hangang.life가 일시적으로 응답을 안 준 경우이며, `errors` 배열에 원인이 찍힙니다. 이 경우 서울시 백업키(`SEOUL_API_KEY`)를 넣어두면 자동으로 백업 소스를 씁니다.
